# Nextdoor DOM Selector Analysis

## Report Indicator Selector

### Problem
The original selector for reported comments used an auto-generated CSS class:
```css
.blocks-19xz1ve
```

This class name is generated by Nextdoor's CSS-in-JS system and will change whenever they:
- Update their styling framework
- Rebuild their CSS bundles
- Deploy new versions of their app

**Result:** The extension would break on every Nextdoor update.

---

## Solution: Stable SVG Icon Selector

### New Selector
```css
svg[data-icon="report-fill"]
```

### Why This Is Stable

1. **Semantic Attribute**: `data-icon="report-fill"` is a semantic identifier that describes _what_ the icon represents, not _how_ it looks
2. **Purposeful**: This attribute is used by Nextdoor's icon system to identify icon types for accessibility and functionality
3. **Unique**: Only appears on reported content (verified in HTML analysis - only 1 match in entire page)
4. **Framework-Independent**: SVG data attributes persist across styling changes, framework updates, and component restructuring

### HTML Structure

The report indicator appears in this DOM hierarchy:

```html
<div data-testid="comment-detail">
  <!-- Other comment content -->
  <div data-testid="comment-detail-body">
    <!-- Comment text -->
  </div>

  <!-- Report indicator (only on flagged comments) -->
  <div class="blocks-19xz1ve">  <!-- ← Auto-generated class (unstable) -->
    <div class="blocks-1ajj0lw">
      <div class="blocks-eug23f">
        <div data-testid="styled-text-wrapper">
          <span data-testid="styled-text">1 report</span>
        </div>
        <svg data-icon="report-fill" data-block="16" aria-hidden="true">
          <!-- ↑ STABLE SELECTOR -->
          <path fill="currentColor" d="..."></path>
        </svg>
      </div>
      <div>
        <!-- "Disrespectful" tag and other details -->
      </div>
    </div>
  </div>
</div>
```

### Selector Strategy

**Primary Selector:** `svg[data-icon="report-fill"]`
- Finds the report icon directly
- Works regardless of parent container class changes

**Usage in Code:**
```javascript
// Find all report indicators
const reportIndicators = document.querySelectorAll('svg[data-icon="report-fill"]');

// Find parent comment container
reportIndicators.forEach((icon) => {
  const commentContainer = icon.closest('[data-testid="comment-detail"]');
  // Process flagged comment...
});
```

## Alternative Selectors (Evaluated but Not Used)

### Option 1: Text Content Matching
```javascript
// Find elements containing "X report(s)" text
Array.from(document.querySelectorAll('[data-testid="styled-text"]'))
  .filter(el => /\d+\s+reports?/.test(el.textContent))
```
**Rejected because:**
- Requires JavaScript text matching (less efficient)
- Could match false positives in other contexts
- Not a pure CSS selector

### Option 2: Parent Container Class
```css
.blocks-19xz1ve  /* Current unstable selector */
```
**Rejected because:**
- Auto-generated by CSS-in-JS framework
- Changes with every build
- This is the problem we're solving!

### Option 3: Has Selector
```css
:has(> svg[data-icon="report-fill"])
```
**Not needed because:**
- We can use `.closest()` in JavaScript to find parent
- SVG selector alone is sufficient for initial detection
- Better browser compatibility with simpler selector

## Maintenance Notes

### When This Selector Might Break

The selector will break if Nextdoor:
1. **Changes icon system** - Replaces `data-icon` with a different attribute system
2. **Removes SVG icons** - Switches to icon fonts or images
3. **Renames icon IDs** - Changes "report-fill" to something else

**Likelihood:** Low
- Icon identifiers are semantic and part of accessibility infrastructure
- Changing them would break internal Nextdoor code and tests
- Much more stable than auto-generated CSS classes

### Monitoring for Changes

If the extension stops finding flagged comments:
1. Check browser console for selector errors
2. Inspect a reported comment in DevTools
3. Look for the report icon and check its attributes
4. Update selector if `data-icon` value changed

### Future Improvements

Consider adding fallback selectors for robustness:
```javascript
const REPORT_SELECTORS = [
  'svg[data-icon="report-fill"]',  // Primary (current)
  'svg[data-icon*="report"]',       // Fallback: any icon with "report" in name
  '[aria-label*="report" i]',       // Fallback: ARIA label
];

// Try each selector until one works
for (const selector of REPORT_SELECTORS) {
  const indicators = document.querySelectorAll(selector);
  if (indicators.length > 0) return indicators;
}
```
